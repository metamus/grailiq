## 2. Stripe — paste env vars into Railway

**Why:** All Stripe code is live (checkout, portal, webhook, paywall
middleware). Pricing page opens Checkout when you click "Start trial".
Without these env vars, `/stripe/checkout` returns 503 and buttons fail.

### Step-by-step

1. Go to **Stripe Dashboard** → <https://dashboard.stripe.com/products>.
   Create two products:
   - **Collector** · recurring · **$9.99/month**
   - **Investor** · recurring · **$24.99/month**

   For each, copy the Price ID that looks like `price_1ABC...xyz`.

2. Go to <https://dashboard.stripe.com/apikeys> and copy your
   **Secret key** (starts with `sk_live_…` or `sk_test_…`).

3. Go to <https://dashboard.stripe.com/webhooks>. Click **Add endpoint**.
   - **Endpoint URL:** `https://grailiq-production.up.railway.app/api/v1/stripe/webhook`
   - **Events to send:**
     - `checkout.session.completed`
     - `customer.subscription.created`
     - `customer.subscription.updated`
     - `customer.subscription.deleted`

   After creating, click the endpoint and reveal the **Signing secret**
   (starts with `whsec_…`).

4. Go to **Railway** → project `grailiq` → service `grailiq-api` →
   **Variables**. Add these four, each on its own line:

   ```
   STRIPE_SECRET_KEY=sk_live_...
   STRIPE_WEBHOOK_SECRET=whsec_...
   STRIPE_PRICE_COLLECTOR=price_...
   STRIPE_PRICE_INVESTOR=price_...
   ```

   Railway auto-redeploys. `GET /api/v1/stripe/subscription` will start
   returning 200 on the next request.

## 3. Supabase — enable Google + Apple OAuth

**Why:** Both web and mobile have OAuth buttons. Without provider creds
configured, the buttons show an error ("google is not enabled in Supabase
Auth"). Email/password works regardless.

### Google

1. **Google Cloud Console** → <https://console.cloud.google.com/apis/credentials>.
2. **Create Credentials** → **OAuth client ID** → Application type
   **Web application**.
3. Authorized redirect URIs (add both):
   ```
   https://ipsgkfonkurdepbrlgwl.supabase.co/auth/v1/callback
   grailiq://auth-callback
   ```
4. Copy the **Client ID** and **Client secret**.
5. Go to **Supabase Dashboard** → `ipsgkfonkurdepbrlgwl` → Authentication
   → Providers → **Google** → enable, paste the Client ID + Client
   secret, save.

### Apple (iOS OAuth)

1. **Apple Developer** → <https://developer.apple.com/account/resources/identifiers/list>.
2. Create a **Services ID** (e.g. `com.grailiq.app.services`) with Sign
   in with Apple enabled.
3. Return URLs:
   ```
   https://ipsgkfonkurdepbrlgwl.supabase.co/auth/v1/callback
   ```
4. Generate a **Key** for Sign in with Apple, download the `.p8` file.
5. In Supabase → Authentication → Providers → **Apple** → enable, paste
   the Services ID, Team ID, Key ID, and Private Key content.

### Supabase redirect URLs

In Supabase → Authentication → URL Configuration, add under **Redirect
URLs**:

```
https://grailiq.com/**
grailiq://auth-callback
```

## 5. Seed retailer mappings

**Why:** `retailer_products` is empty. Without rows, the restock worker
logs `mappingsMissing` and skips. With rows, it starts polling every 60
seconds.

### Option A — use the admin UI (fastest for a handful)

1. Sign in at <https://grailiq.com/sign-in>.
2. Navigate to `/app/admin/retailers` (or type the URL directly — not in
   the sidebar yet, on purpose).
3. Click **Add mapping**. Paste a product UUID (from `/app/sets` → pick
   a set → product detail page URL `/app/products/<uuid>`), pick a
   retailer, paste a URL, paste a SKU/TCIN if the retailer needs one
   (Target needs TCIN, Best Buy needs SKU).

### Option B — bulk JSON

1. Edit `grailiq-api/src/scripts/starter-mappings.json` with real URLs.
2. Run from repo root:
   ```bash
   cd grailiq-api
   npx tsx src/scripts/seed-retailer-mappings.ts src/scripts/starter-mappings.json
   ```

### Retailer-specific formats

- **Pokémon Center:** `https://www.pokemoncenter.com/product/<id>/...` —
  URL alone is enough, no SKU needed. May hit Akamai bot detection in
  prod; proxy setup is a follow-up.
- **Target:** TCIN is the 8-digit number in the URL:
  `https://www.target.com/p/.../A-93954435` → SKU = `93954435`.
- **Best Buy:** SKU in the URL: `/site/.../6539847.p?skuId=6539847`.
- **Amazon:** ASIN (10-char) — stub only, won't actually work yet.
- **Walmart:** URL alone works most of the time; no SKU needed.

## 6. Resend domain — for transactional email

**Why:** Without this, restock alerts and the weekly digest log only and
don't ship.

1. <https://resend.com/domains> → Add `grailiq.com`.
2. Add the DNS records Resend shows (SPF + DKIM TXT, DMARC) to
   Namecheap's DNS for grailiq.com.
3. Wait for verification (a few minutes).
4. <https://resend.com/api-keys> → create an API key.
5. Railway → `grailiq-api` → Variables:
   ```
   RESEND_API_KEY=re_...
   ```
6. From address is already hardcoded: `alerts@grailiq.com` for restocks,
   `weekly@grailiq.com` for digests. Make sure those aliases are
   authorized in Resend.
